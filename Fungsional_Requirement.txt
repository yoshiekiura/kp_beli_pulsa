Pembelian Pulsa

Fungsional Requirement
keterangan : tb = tidak bisa, sb = sudah bisa, bg = belom garap, otw = perjalanan

	Pelanggan
•	Pelanggan dapat membeli pulsa tanpa mendaftar dahulu (tb)
•	Pelanggan dapat mendaftar (sb)
•	Pelanggan dapat login (sb)
•	Pelanggan dapat melihat profil setelah login (sb)
•	Pelanggan dapat mengedit profil setelah login (sb)
•	Pelanggan melihat riwayat transaksi setelah login (sb*belom ada database)
•	Pelanggan dapat mengisi saldo setelah login (tb)
•	Pelanggan dapat melihat mutasi saldo setelah login (tb)
•	Pelanggan dapat mengganti password setelah login (sb)
•	Pelanggan dapat melihat harga pulsa setelah login/sebelum (sb)
•	Pelanggan dapat menulis testimoni setelah pembayaran telah sukses (tb)
•	Pelanggan dapat melihat testimoni tanpa login (sb)
•	Pelanggan dapat print rincian transaksi (sb)
•	Pelanggan dapat melihat rincian transaksi (tb)
•	Pelanggan dapat mencari rincian transaksi (tb)
•	Pelanggan dapat menverifikasi akun melalui email (sb)
•	Pelanggan dapat melakukan complain saat pembelian pulsa (tb)

	Admin
•	Admin dapat melihat transaksi pembelian (bg)
•	Admin dapat melihat pelanggan yang sudah terdaftar (sb)
•	Admin dapat melihat complain para pelanggan (bg)
•	Admin dapat memproses pembelian para pelanggan semisal status belum bayar, Proses, Sukses, gagal. (bg)
